MASTER GIT AND GITHUB

1. Overview of git and github

                
                                 
                          
    
        
  • Git is a version control system to keep track of changes to files and projects over time. Git is used for
managing the changes to a project over time. A project might be just a single file, a handful of files,or
thousands of files. Those files can be anything from plain text to images or videos.

Because Git is focused on managing changes, it is often used as a collaboration tool allowing people to work
on the same project at the same time. By tracking their individual changes, Git can bring everything together
to the final version.
        
  • Github is a website that hosts Git repositories online, making it easier for developers to share code. It allows
you to work collaboratively with other people around the world, plan your projects and track your work.

3. Some basic commands

  • Config command: The git config command is a convenience function that is used to set Git configuration values on a global or local project level. It sets the author name and email address respectively to be used with your commits.
    Ex: git config -global user.email "your email"
    git confit -global user.name "your user name"
  • Add command: This command adds a file to the staging area.
    Ex: git add [file]
  • Commit command: This command records or snapshots the file permanently in the version history.
    Ex: git commit -m "the commmit message"
  • Push command: This command sends the committed changes of master branch to your remote repository.
    Ex: git push [variable name] master

4. How to resolve conflicts

  • When a conflict occurs, you can merge branches manually. Let's merge the branch that you're working on and which is the target branch for the pull request to be pulled.
  • There are 3 steps to resolve conflict:
    - Pull master branch.
    - Resolve the conflict locally.
    - Commit and push the modified source code again.

5. Branch

  • In the latest version of Git, you can simply create a branch from the remote repository on your local repository by assigning an existing branch name in the remote repository when calling Git checkout.
    Ex: git checkout [branch]
  • If you are on an older version of Git, you can do so with the command below.
    Ex: git branch [branchname] origin/[branch]
